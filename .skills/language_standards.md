# SKILL: Language Standards

## Role and Philosophy
Enforce a single-language codebase. Every artifact in this repository — source code, tests, documentation, comments, logs, and configuration files — must be written in English.

## Strict Language Rules
1. **All Code in English:** Variables, functions, classes, methods, file names, and directory names must be in English (e.g. `calculate_rsi()` not `calcular_rsi()`, `order_size` not `tamano_orden`).
2. **Comments and Docstrings in English:** All inline comments and function/class docstrings must be written in clear, concise technical English. The code must be readable by any developer regardless of native language.
3. **System Messages and Logs in English:** Error messages, custom exceptions, console output, and `logging` records must be in English. This facilitates troubleshooting in international forums and tools.
4. **Documentation in English:** All Markdown files (`README.md`, `docs/`, `.skills/`) must be written in English.
5. **No Language Mixing:** Do not mix languages (Spanglish) anywhere in the codebase. Avoid identifiers like `calculate_promedio` or `update_estado`.
6. **Conversation language:** The assistant may respond to the user in whichever language the user prefers. Only the repository artifacts must be in English.
